Advanced Settings
Search Mode
How the AI navigates your lorebook tree during generation.
Step-by-step navigation. AI sees one level at a time, picks a branch, drills deeper. Uses multiple tool calls. Better for very large trees (100+ nodes) where showing everything at once would overwhelm context.
Full tree shown at once. AI sees all categories and summaries in a single view and picks directly. Uses fewer tool calls, better retrieval accuracy. Based on RAPTOR research showing collapsed-tree consistently outperforms top-down traversal.
Collapsed Tree Depth
How many levels of the tree to show in the tool description. The AI sees this many levels at once and navigates deeper with additional tool calls. Lower values = smaller tool description but more calls needed. Higher values = more visible at once but heavier on context.
levels visible
1 — Top-level only. Minimal context, always needs navigate calls.
2 — Two levels. Good balance for most lorebooks. Default
3-4 — Deep view. Better for small-medium lorebooks where you want single-call retrieval.
Selective Retrieval
Off (default): When the AI reaches a leaf node, all entries are injected into context automatically. Simpler, fewer tool calls.
On: When the AI reaches a leaf node, it sees a manifest of entry titles/keys instead of full content. It must then request specific entries by name. More precise retrieval — useful for large leaf nodes where only some entries are relevant — but costs an extra tool call per retrieval.
Tool Call Recursion Limit
Maximum number of consecutive tool calls the AI can make per generation. SillyTavern's default is 5. Raising this allows deeper tree traversal and more tool actions per turn, but costs more API tokens. Most relevant in Traversal mode.
calls per generation
Values above 10 may significantly increase API costs. Each recursion sends the full context + tool results back to the API.
Prompt Injection
Controls where and how TunnelVision injects its prompts into the conversation. Position and depth determine where the prompt lands in the message list the AI sees.
Injects instructions explaining the memory system to the AI — when to use tools vs. rely on auto-injected context. Customize the text below to match your workflow.
Notebook Injection
Where the AI's private notebook notes are injected into context each turn.
In Chat injects at a specific depth in the message list (0 = bottom, 1 = before last message). Better for model attention on long chats.
In System Prompt places it in the story string area (depth is ignored). More stable but can get buried in long contexts.
Role controls which role the injected message appears as.
Total Injection Budget
Caps the combined size of all TunnelVision context injections (memory system prompt + world state + smart context + notebook). Use this to control how much of your context window TunnelVision consumes. Set to 0 for unlimited (each injection is governed only by its own per-feature limit).
characters (0 = unlimited)
Rough guide: 1000 chars ≈ 250 tokens. A budget of 8000 (~2000 tokens) is a sensible starting point. Priority order when trimming: memory system prompt > world state > smart context > notebook.
Hide Tool-Call Messages
When enabled, TunnelVision tool-call messages are visually hidden in the chat. The AI still uses them normally, but the messages won't appear in the conversation. Useful for a cleaner chat experience.
Ephemeral Tool Results
Strip retrieved lorebook content from older tool-call messages before each generation. This prevents past search results from accumulating in context and bloating your chat history.
Warning: This permanently modifies chat message data. Old tool results will be replaced with a short stub and cannot be recovered. The AI will re-search for lore each turn instead of seeing past retrievals. This saves context tokens but costs more tool calls.
Choose which tool results get stripped. Notebook is always protected.
LLM Build Detail Level
How much entry data to send when building/summarizing the tree with LLM. Applies to categorization, subdivision, and summary generation.
Full sends complete untruncated entry content — best categorization but heavy on tokens. Use with large context models (80k+).
Lite sends a 150-char content preview + keys, groups, and settings. Good balance of quality and cost. Default
Names — cheapest, AI can only group by titles.
Tree Granularity
Controls how aggressively the LLM breaks up your entries during tree building. Higher = more categories, more sub-categories, fewer entries per node. Lower = broader groupings with more entries per node.
Auto scales splitting based on how many entries you have. Override if you want the tree broken up more or less than the default. Requires a tree rebuild to take effect.
LLM Chunk Size
Maximum characters per LLM call when categorizing entries. Large lorebooks are split into chunks of this size. The last entry in a chunk is always included in full (overfill) so entries are never cut off mid-way. Lower values = more LLM calls but safer for smaller context models.
characters (~7,500 tokens at default)
30,000 chars is a safe default for most APIs (~8k tokens). Raise to 100,000+ for large-context models (128k+). If your lorebook fits in one chunk, no splitting occurs.
Duplicate Detection
When enabled, the Remember tool checks for semantically similar existing entries before saving. If a near-duplicate is found, the AI is warned (but not blocked) — it can choose to update the existing entry instead.

Smart Context (Proactive Pre-fetch)
Automatically injects relevant lorebook entries into context before each generation — no Search tool call needed. Matches entity names and keywords from recent messages against entry titles and keys.
messages
Controls how much context is auto-injected. Higher values provide more information but consume more of the context window.
Constant Entry Passthrough
Lorebook entries marked as "Constant" (always active) are normally suppressed along with all other TV-managed entries. Enable this to let constant entries pass through ST's normal injection pipeline, so they're always included in context regardless of tree search.
Useful for chat summaries, system rules, or other entries that should always be in context. These entries will still appear in the tree but won't depend on tool calls for injection.
Multi-Lorebook Mode
How to present multiple active lorebooks to the AI during search.
All lorebook trees merged into one view. AI sees all categories as siblings. Fewer tool calls, simpler navigation.
Each lorebook shown as a separate document node. AI picks a lorebook first, then navigates its tree. Better for very different lorebooks.
Connection Profile
Optional: Select a connection profile to use for TV's LLM operations (tree building, summaries, chat ingest). If set, TV will switch to this profile before LLM calls and restore the original after. Leave empty to use the current API settings.
Background Model (Sidecar)
A separate, typically cheaper LLM for background operations (smart context reranking, analytical tasks). When enabled, background work is routed here instead of the main chat API.
Cost: ~$0.001–0.005 per background cycle with a cheap model.
Embedding Model
A separate endpoint for computing text embeddings (used by Smart Context for semantic similarity). Can point to a different proxy or provider than the sidecar or main API.
Cost: ~$0.02/million tokens (OpenAI), free for local models. Cached per session.
Diagnostics

Import / Export
Export
Download data from TunnelVision in portable formats. Exports use data from the currently active chat and selected lorebook.
Import Facts
Import facts from a plain text or Markdown file. Each non-empty line becomes a new lorebook entry in the selected lorebook. Lines starting with # are skipped as comments. Lines in the format Title: Content will use the text before the colon as the entry title.
